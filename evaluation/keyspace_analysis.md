# Key Space & Cryptographic Security Analysis

This document provides a rigorous theoretical evaluation of the key space and security profiles of both the **Robust Mode** and the **Paper Mode** in the fractional-order hyperchaotic synchronization secure communication framework.

---

## 1. Robust Mode Analysis

### SHA-256 Seed Space
The Robust Mode uses a SHA-256 hash function to compress the extracted bits into a 256-bit seed.
* **Seed Size**: 256 bits.
* **Theoretical Keyspace**: $2^{256} \approx 1.15 \times 10^{77}$ combinations.
* **Brute-Force Resistance**: With a complexity of $2^{256}$, Robust Mode is completely immune to brute-force attacks by modern or quantum computers (which would require $2^{128}$ operations using Grover's algorithm).

### Entropy Extraction Impact
* **Sign-Based Majority Voting**: The raw trajectory states are processed in blocks of size 32. For an image of size $256 \times 256$ ($65,536$ pixels), the trajectory has 2,048 blocks.
* **Raw Bit Extraction**: Across 5 stable states ($x_2$ to $x_6$), a total of $5 \times 2048 = 10,240$ bits are extracted.
* **Information Compression**: These $10,240$ bits contain spatial correlations and redundancies inherent to the chaotic attractor. Passing them through SHA-256 compresses and diffuses the entropy, eliminating statistical biases and producing a cryptographically uniform 256-bit seed.

### Effective Keyspace
The effective key space is determined by the combination of:
1. The 6D initial conditions $x_0$ (each a 64-bit float, giving $\approx 276$ bits of entropy).
2. The fractional order $q$ and system parameters $a, b$.
3. The SHA-256 seed ($256$ bits) which seeds the `numpy.random.default_rng` generator.
The PRNG expands the 256-bit seed into a keystream of size $N \times 6$ with uniform distribution over $[0, 255]$, yielding an effective keyspace of $2^{256}$ keys.

## 2. Paper Mode Analysis

### Quantization Space
In Paper Mode, keys are generated by directly quantizing the trajectory states:
$$\text{key}_i = \lfloor |state| \times S \rfloor \pmod{256}$$
* **State Selection**: To ensure 100% matching key sequences between the transmitter and receiver under nominal synchronization error ($\approx 1.5 \times 10^{-3}$), a coarse quantization combination is used:
  $$c_i = (q_1[i] \cdot 27 + q_3[i] \cdot 9 + q_5[i] \cdot 3 + q_6[i]) \pmod{256}$$
  where $q_1, q_3, q_5$ are quantized at scale $0.4$ (taking values in $\{0, 1, 2\}$) and $q_6$ at scale $0.1$ (taking values in $\{0, 1\}$).
* **Key Cardinality**: This algebraic combination theoretically allows $3^3 \times 2 = 54$ possible key values. Due to the bounded nature of the hyperchaotic attractor, only **17 unique key values** are actually observed in practice.
* **Entropy Deficit**: Because the key sequence contains only 17 unique values instead of 256, it exhibits a massive entropy deficit. The local randomness is low, leaving the cipher vulnerable to statistical and ciphertext-only attacks.

### State Sensitivity
* **Chaotic Divergence**: The fractional-order system exhibits strong sensitivity to initial conditions. A perturbation of $10^{-14}$ in any initial state grows exponentially over the simulation window, leading to complete divergence of the trajectory.
* **Keystream Sensitivity**: Because the trajectory diverges, the resulting coarse-quantized key sequence changes completely. This ensures that the cipher is highly sensitive to the exact initial conditions and parameters.

### Effective Keyspace
* Since there is no PRNG expansion, the security of the cipher rests entirely on the secrecy of the physical parameters:
  * 6 initial states ($x_1(0) \dots x_6(0)$): $\approx 276$ bits of entropy.
  * System parameters $a, b, q$: $\approx 100$ bits of entropy.
* **Theoretical Keyspace**: $\approx 2^{376}$.
* **Cryptographic Vulnerability**: Although the theoretical keyspace is large, the *keystream* itself is weak. The direct mapping of states to keys without cryptographic post-processing allows an attacker to reconstruct the phase portrait of the attractor from the ciphertext if the permutation is compromised, leading to complete recovery of the plaintext.

## 3. Comparative Summary

| Security Aspect | Robust Mode | Paper Mode |
| :--- | :---: | :---: |
| **Keystream Distribution** | Uniformly distributed $[0, 255]$ | Discrete sparse values (17 unique values) |
| **Keystream Entropy** | Near-ideal ($\approx 7.99$) | Lower ($\approx 7.96$) |
| **Brute-Force Resistance** | High ($2^{256}$) | High theoretically ($2^{376}$), low practically |
| **Statistical Security** | High (avalanche propagation) | Medium (no PRNG diffusion expansion) |
| **Synchronization Dependency** | Low (block-based tolerance) | High (extremely sensitive to state noise) |
